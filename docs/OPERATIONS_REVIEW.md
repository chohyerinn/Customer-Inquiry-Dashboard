# 고객 문의 운영 리뷰 샘플

이 문서는 쇼핑몰 고객 문의 운영자가 하루 단위로 확인할 만한 항목을 정리한 샘플입니다. 실제 고객 데이터를 사용하지 않았고, 아래 발화와 수치는 모두 포트폴리오 설명을 위한 가상 예시입니다.

프로젝트의 핵심은 단순히 답변을 자동 생성하는 것이 아니라, 자동화가 실수할 수 있는 지점을 운영자가 어떻게 확인하고 다음 개선으로 넘길지 정리하는 데 있습니다.

## Daily review checklist

- 신규 문의 중 개인정보가 포함된 건이 마스킹되었는지 확인한다.
- 자동 분류 결과가 실제 문의 의도와 맞는지 샘플링한다.
- 폴백 처리된 문의가 반복되는 표현인지 확인한다.
- 상담원 연결이 필요한 케이스가 일반 큐로 들어가지 않았는지 확인한다.
- 다음 학습 후보로 추가할 발화를 정리한다.
- 데모 데이터 또는 외부 API 실패로 인한 폴백이 운영 화면에서 구분되는지 확인한다.

## Review cases

| User message | Predicted result | Review result | Action |
|---|---|---|---|
| 카드 결제했는데 주문내역이 없어요 | coupon_issue | payment_error | 결제 오류 표현으로 재분류하고 상담원 확인 대상으로 표시 |
| 배송지 바꿨는데 예전 주소로 간 것 같아요 | fallback | address_change + delivery_status | 복합 의도 케이스로 기록하고 배송/주소 변경 발화 후보에 추가 |
| 제 전화번호는 010-0000-0000인데 연락 주세요 | privacy_included | privacy_included | 전화번호 마스킹 후 외부 LLM 호출 차단 여부 확인 |
| 환불 신청했는데 아직 입금이 안 됐어요 | refund_request | refund_delay | 환불 요청과 환불 지연 문의를 분리할 필요가 있음 |
| 상담원이랑 바로 얘기하고 싶어요 | general | human_handoff | 상담원 연결 의도가 명확한 표현으로 우선순위 조정 |

## Fallback review

폴백은 단순 실패로만 보지 않고, 다음 세 가지로 나눠서 봅니다.

1. **표현 부족**: 같은 의미인데 기존 규칙에 없는 말투가 들어온 경우
2. **복합 의도**: 배송지 변경과 배송 조회처럼 두 가지 의도가 섞인 경우
3. **외부 서비스 실패**: CLOVA Studio 또는 내부 API 응답이 실패해 템플릿 응답으로 대체된 경우

이렇게 나누면 폴백 건수를 줄이는 작업이 단순히 규칙을 늘리는 일이 아니라, 데이터 보강·시나리오 분리·장애 대응 중 어디에 가까운지 판단할 수 있습니다.

## Training utterance candidates

아래 표현은 바로 정답 데이터로 넣기보다, 운영자가 검토한 뒤 추가하는 후보로 둡니다.

| Candidate utterance | Suggested intent | Why it matters |
|---|---|---|
| 주문은 됐는데 결제 문자만 왔어요 | payment_error | 결제 완료와 주문 생성 실패가 섞인 표현 |
| 배송지가 예전 집으로 되어 있어요 | address_change | 배송 조회가 아니라 주소 변경 이슈일 수 있음 |
| 환불 언제 들어와요 | refund_delay | 환불 요청과 환불 지연 문의를 구분해야 함 |
| 개인정보 지워주세요 | privacy_request | 개인정보 처리 관련 별도 응대가 필요함 |

## Release review notes

새 분류 규칙이나 응답 템플릿을 반영하기 전에는 아래 항목을 확인합니다.

- 새 intent가 기존 intent와 의미상 겹치지 않는지 확인
- 고위험 문의가 자동 응답만 받고 끝나지 않는지 확인
- 개인정보 포함 문의가 외부 LLM으로 전달되지 않는지 확인
- fallback 문구가 고객에게 너무 차갑게 보이지 않는지 확인
- 대시보드의 fallback, handoff, PII count가 정상 집계되는지 확인

이 문서는 실제 운영 리포트가 아니라, 고객 문의 자동화 프로젝트에서 어떤 항목을 모니터링해야 하는지 정리한 샘플입니다.
