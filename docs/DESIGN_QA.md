## 세부 메모

### ✅ 잘 된 것

- **처리 중 단계의 단계 표시**: `개인정보 탐지·마스킹 → AI 초안 생성 → 상담원 자동 배정` 3단계 진행 애니메이션이 BRIEF의 발표 흐름과 정확히 일치
- **오류 복구 경로**: 빨간 배너 + "다시 시도" 버튼 + 고객센터 연락처 링크 모두 구현, 입력값 보존(`initial` prop 전달)
- **CLOVA 오류 상태 전용 화면**: `dashboard · CLOVA 오류` 화면이 별도로 있어 템플릿 폴백 시나리오를 발표에서 시연 가능
- **PII 마스킹 시연**: 결과 화면에 "탐지·마스킹된 개인정보" 카드가 원본 타입 → 마스킹값 매핑으로 표시 — AI 기능의 핵심을 시각화
- **더미 데이터 안전성**: `data.js` 주석에 "실제 개인정보/금융정보/키 없음" 명시, 상담원도 "가상 인물(실제 아님)" 표기

### ⚠️ 확인 필요

- **모바일 프리뷰 공백**: `_preview.png`에서 폰 프레임 내부가 비어있음. 폰트 파일(woff2) 로딩 완료 전 캡처된 것일 가능성이 있으나, 실제 `index.html`을 브라우저에서 열어 390px 뷰포트로 확인해야 함
- **`/admin/counselors` 화면 없음**: BRIEF에 명시된 5개 화면 중 상담원 관리(`/admin/counselors`)가 구현되어 있지 않음 — 발표 흐름에 이 화면이 필요한지 확인 필요
